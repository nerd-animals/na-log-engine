이번 포스팅에서는 **NA-Log에서 Image 사용 방법**에 대해 알려드리겠습니다.

----

### 전반적인 folder 구조는 다음과 같습니다.

```bash
--| .github/
--| na-log-engine/ <--- submodule로 저희가 개발/유지보수하는 NA-Log 엔진입니다.
--| posts/  <--- 사용자의 게시글이 저장되는 공간입니다. 게시글은 `mdx` 확장자입니다.
--| public/ <--- 게시글 외, 자원을 저장하는 공간입니다. 이 곳에 이미지를 저장하면 됩니다.
```

게시글에서 `public`에 있는 이미지를 사용하고 싶다면, `public/`을 기본 경로라고 생각하시면 됩니다.

예를 들어, 아래와 같이 `public/crazy-gugu.jpg`가 있다고 가정하겠습니다.

```bash
--| public/
----| crazy-gugu.jpg
```

---- 

### 1. Markdown 문법 사용하기

NA-Log는 Github-Flavored-Markdown 문법을 지원합니다. \
따라서, 아래와 같이 사용할 수 있습니다. 

#### Public 폴더 내에 있는 이미지 사용
```bash
![비둘기?](/crazy-gugu.jpg)
![비둘기?](crazy-gugu.jpg) # /가 없어도 잘 첨부됩니다!
```

![비둘기?](/crazy-gugu.jpg)

![비둘기?](crazy-gugu.jpg)

#### Web에 있는 이미지 사용
```bash
# web에 있는 이미지를 사용할 때,
![비둘기?](https://github.com/nerd-animals/na-log/assets/56672129/ad8d1f15-9b4e-456f-afc1-bb922faf1e9d)
```
![비둘기?](https://github.com/nerd-animals/na-log/assets/56672129/ad8d1f15-9b4e-456f-afc1-bb922faf1e9d)

### 2. HTML 태그 사용하기 (비권장 😥) 

GFM에서 HTML 태그를 사용할 수 있으므로 `img`태그를 사용한 이미지 첨부가 가능합니다.
이미지 경로에 `basePath`를 적용해주어야 하는데, HTML 태그로 입력할 경우 처리가 안 됩니다.

그러나 세부 옵션을 조정하려면, HTML 태그를 사용해야 합니다.
HTML 태그를 사용할 경우, basePath를 직접 추가해주세요!
basePath는 본인의 블로그 링크에서 확인해주시면 됩니다.
`<Github Owner>.github.io/<basePath>`

```bash
<img src="/na-log/crazy-gugu.jpg" alt="비둘기?" />
```

<img src="/na-log/crazy-gugu.jpg" alt="비둘기?" />
