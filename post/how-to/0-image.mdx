### 안녕하세요, Nerd-Animals의 raccoon입니다 🦝

블로그 포스팅에서 이미지/움짤(?) 첨부는 필수입니다. \
이번 포스팅에서는 저희가 사용 중인 **Image 첨부 방법**에 대해 소개합니다 😀

----

## 파일은 어디에 저장하나요?

파일은 Github에 업로드해서 사용합니다. \
Github에 무료로 파일을 저장할 수 있고, 빠르게 이미지 링크를 얻을 수 있습니다.

![Github에서 이미지 링크 얻기](https://github.com/nerd-animals/na-log-engine/assets/56672129/32024890-28f1-4207-8cb9-6a059da5cd8e)

1. `New Issue`를 이용해서 Text Box를 켭니다.(Issue가 아니어도 됩니다.)
2. 파일을 Drag & Drop 합니다. (이미지 파일은 Copy & Paste도 가능합니다!)
3. 생성된 링크를 사용합니다! 🎉

---- 

## 생성된 링크를 어떻게 사용하나요?

### 1. Markdown 문법 사용하기

NA-Log는 Github-Flavored-Markdown 문법을 지원합니다. \
Markdown 문법을 사용하면 간단하게 이미지를 첨부할 수 있습니다.

```bash
![비둘기..?](https://github.com/nerd-animals/na-log-engine/assets/56672129/e4ea21ae-0d7b-49cc-a0b2-517e73925134) # ![alt](src)
```

![비둘기?](https://github.com/nerd-animals/na-log-engine/assets/56672129/e4ea21ae-0d7b-49cc-a0b2-517e73925134)

### 2. \<img> 태그 사용하기

\<img> 태그를 사용해서 이미지를 첨부할 수 있습니다. \
Markdown 문법보다 더욱 다양한 옵션을 사용할 수 있습니다. 관련 정보는 [MDN Web Docs](https://developer.mozilla.org/ko/docs/Web/HTML/Element/img)를 참고하세요 😊

```bash
<img src="https://github.com/nerd-animals/na-log-engine/assets/56672129/e4ea21ae-0d7b-49cc-a0b2-517e73925134" alt="비둘기?" />
```
<img src="https://github.com/nerd-animals/na-log-engine/assets/56672129/e4ea21ae-0d7b-49cc-a0b2-517e73925134" alt="비둘기?" />
----
